# MemoTask 设计规划快速理解版

生成日期：2026-06-22  
当前版本：Memo 容器模型 / 第一版无日期系统，后续支持月份地图  
目的：用更短、更清晰的方式说明 MemoTask 的设计核心，帮助快速理解产品应该怎么用、怎么展示、先做什么。

## 1. 一句话说明

MemoTask 是一个跨手机和 PC 的个人想法整理工具：你随手写下一段 Memo，AI 在这个 Memo 的包围框内部生成待办事项，你通过拖动整个 Memo 的前后顺序来表达紧急程度。

它的第一版核心不是“按日期管理任务”，而是：

> 每个 Memo 是一个任务包，Memo 的排序就是优先级。

## 2. 为什么这样设计

传统待办软件通常依赖日期、截止时间、提醒和今日列表。但对自用工具来说，日期系统有一个明显副作用：

- 没按日期完成时容易产生挫败感。
- 过期任务会越积越多。
- 用户会开始逃避打开软件。
- 工具从“帮助完成”变成“制造压力”。

所以 MemoTask 改成更轻的模式：

```text
不问什么时候必须完成
只问现在哪个 Memo 更重要
```

你只需要把更紧急、更想先处理的 Memo 拖到前面。越靠前，越应该优先看；越靠后，就暂时放着。

## 3. 产品核心闭环

MemoTask 的主流程变成四步：

```text
写下 Memo -> AI 在 Memo 内生成待办 -> 勾选/编辑待办 -> 拖动 Memo 排优先级
```

### 第一步：写下 Memo

你可以像写便签一样输入：

```text
研究一下 PWA 能不能覆盖手机和 PC，然后整理 MemoTask 的实现方案。顺便看看有没有更适合拖拽排序的交互库。
```

系统先保存完整原文，形成一个大的 Memo 包围框。

### 第二步：AI 在 Memo 内生成待办

AI 不会把任务拆到外部列表，而是在这个 Memo 框内部生成小的待办项：

```text
┌────────────────────────────────────────┐
│ Memo：研究 PWA 和拖拽实现方案           │
│                                        │
│ 原文：研究一下 PWA 能不能覆盖手机和 PC... │
│                                        │
│ AI 待办：                               │
│ ☐ 调研 PWA 对手机和 PC 的支持情况        │
│ ☐ 整理 MemoTask 的实现方案              │
│ ☐ 查找适合拖拽排序的交互库              │
└────────────────────────────────────────┘
```

### 第三步：处理 Memo 内部待办

你可以在 Memo 内：

- 勾选完成某个待办。
- 编辑 AI 生成的待办文本。
- 删除不需要的待办。
- 手动新增待办。
- 把待办标记为暂不处理。

这里没有“转成独立任务”的强制步骤。Memo 本身就是任务容器。

### 第四步：拖动 Memo 排优先级

整个 Memo 框可以上下拖动：

```text
1. 最前面的 Memo：当前最紧急
2. 第二个 Memo：次紧急
3. 后面的 Memo：以后再处理
```

优先级不是字段，也不是日期，而是位置。

## 4. 新的信息结构

新版模型只有两个核心对象：

```text
Memo
  └── MemoTodo
```

### Memo

Memo 是一个大的包围框，也是主要操作单位。

它包含：

- 原始想法文本。
- AI 摘要。
- 内部待办列表。
- 当前排序位置。
- 完成进度。
- 标签或分类，可选。
- 创建时间和更新时间，仅用于记录，不用于任务规划。

### MemoTodo

MemoTodo 是 Memo 内部的一条待办。

它包含：

- 待办标题。
- 是否完成。
- 是否由 AI 生成。
- 排序位置。
- 置信度，可选。

关键变化：

```text
旧模型：Memo -> CandidateTask -> Task -> Today/Upcoming
新模型：Memo -> MemoTodo，Memo 排序表示紧急程度
```

## 5. 页面规划

### 5.1 手机端

手机端只需要三个主入口：

```text
Capture  记录
Memos    Memo 队列
Search   搜索
```

手机首页建议直接显示：

- 顶部快速输入框。
- 下方 Memo 卡片队列。
- 每个 Memo 卡片展示前 3 条内部待办。
- 支持长按拖动 Memo 排序。

### 5.2 PC 端

PC 端可以采用更宽的布局：

```text
左侧：快速记录 + 搜索
中间：Memo 卡片队列
右侧：选中 Memo 的完整内容和待办编辑
```

PC 端重点不是日历视图，而是更舒服地整理 Memo：

- 拖动 Memo 排序。
- 展开/折叠 Memo。
- 批量整理 AI 待办。
- 搜索旧 Memo。

## 6. Memo 卡片设计

每个 Memo 都是一个明显的包围框。

卡片建议包含：

```text
┌────────────────────────────────────────┐
│ 拖拽手柄  Memo 标题 / 摘要              │
│ 原始内容预览                            │
│                                        │
│ AI 待办                                 │
│ ☐ 第一条待办                            │
│ ☐ 第二条待办                            │
│ ☐ 第三条待办                            │
│                                        │
│ 进度：1/3 已完成                        │
└────────────────────────────────────────┘
```

交互规则：

- 卡片整体可以拖动。
- 点击卡片可以展开。
- 勾选内部待办不会改变 Memo 顺序。
- 全部待办完成后，Memo 可以标记为完成或归档。
- 未完成的 Memo 始终保留在队列中。

## 7. 第一版不需要日期系统，后续加入月份地图

第一版不做：

- 截止日期。
- 今日任务。
- 未来任务。
- 日历视图。
- 到期提醒。
- 过期任务。
- 按日期滚动任务。

保留的时间字段只用于系统记录：

- Memo 创建时间。
- Memo 更新时间。
- 待办完成时间。

这些字段用于搜索、排序恢复和历史记录，不用于给用户制造计划压力。

后续版本会加入日期板块，但它不是第一版核心。日期板块的形态是“完整月份地图”：展示一个月的所有日期，每个日期格子里显示关联的待办事项。添加方式不是在全局日历里新建任务，而是在 Memo 内部的待办项上开放日期接口，由用户手动把某条 MemoTodo 关联到某一天。

## 8. 优先级如何表达

优先级由 Memo 的顺序表达。

```text
越靠前 = 越紧急 / 越想先处理
越靠后 = 暂时不急 / 以后再说
```

这比 P1/P2/P3 或日期更轻：

- 不需要填写字段。
- 不需要纠结日期。
- 不会因为没完成而产生过期负担。
- 每次只要问自己：“这个 Memo 要不要往前拖？”

## 9. MVP 必做功能

首版只做这些：

1. 手机和 PC 都能打开使用。
2. 支持快速创建 Memo。
3. Memo 以大卡片包围框展示。
4. AI 在 Memo 内生成待办项。
5. 用户能编辑、勾选、删除、新增 Memo 内待办。
6. 用户能拖动 Memo 卡片调整前后顺序。
7. Memo 顺序能跨手机和 PC 同步。
8. Memo 可展开、折叠、归档。
9. 支持搜索 Memo 原文和内部待办。
10. 支持数据导出。
11. 不出现任何付费、会员、订阅或升级入口。
12. 第一版不出现日期规划、Today、Upcoming、日历或过期任务机制。

一句话 MVP：

> 打开就能写 Memo，AI 在框内生成待办，你拖动 Memo 排紧急程度，手机和 PC 顺序一致。

## 10. 后续增强功能

### P1：增强日常可用性

- PWA 安装。
- 离线创建 Memo。
- 手机分享入口。
- 语音输入。
- PC 快捷键输入。
- Memo 标签。
- Memo 归档。
- Memo 完成进度。
- 拖拽排序冲突处理。

### P2：增强自动化和数据控制

- 浏览器扩展。
- 图片 OCR。
- 自然语言搜索。
- AI 周报，但不生成日期计划。
- 月份地图日期板块：一个完整月视图，每个日期显示关联待办。
- MemoTodo 日期接口：在 Memo 内部待办上手动设置所属日期。
- 自定义 AI 提取规则。
- 完整本地优先同步。
- 端到端加密。
- 开放 API。

## 11. AI 的工作边界

AI 应该做：

- 总结 Memo。
- 在 Memo 内提取待办项。
- 建议拆分复杂待办。
- 识别哪些句子只是背景信息。
- 给出低干扰的整理建议。

AI 不应该做：

- 把待办移出 Memo 容器。
- 自动创建外部任务。
- 第一版自动安排日期。
- 自动改变 Memo 排序。
- 替你决定哪个 Memo 更紧急。

AI 只整理内容，不管理你的时间。

## 12. 同步策略

同步重点从“任务日期”变成“Memo 队列顺序”。

MVP 同步内容：

- Memo 内容。
- Memo 内部待办。
- Memo 完成状态。
- Memo 归档状态。
- Memo 排序位置。

排序同步建议：

- 每个 Memo 保存一个 `sort_order`。
- 拖动后更新受影响 Memo 的排序值。
- 多端同时拖动时，以最后一次拖动为准。
- 后续可以加入冲突提示，但首版不需要复杂合并。

## 13. 判断是否做对的标准

如果 MVP 做对了，应该满足：

1. 你能在 3 秒内写下一条 Memo。
2. Memo 看起来像一个清晰的大包围框。
3. AI 生成的待办明确属于这个 Memo，而不是散落到外面。
4. 你能勾选 Memo 内部待办。
5. 你能拖动 Memo 调整优先级。
6. 手机和 PC 的 Memo 顺序一致。
7. 没有日期、过期、今日清单带来的心理压力。
8. 打开软件时，你只需要看最上面的几个 Memo。

最关键的成功标准是：

> 它让你更愿意继续处理事情，而不是因为任务过期而逃避。

## 14. 最终设计核心

MemoTask 的新版设计核心是三句话：

1. **Memo 是容器**：每个 Memo 是一个大包围框，内部包含 AI 生成的待办。
2. **顺序是优先级**：拖动 Memo 的前后顺序，就表示任务紧急程度。
3. **第一版不要日期压力**：不做 Today、Upcoming、截止日期和过期任务；后续日期板块只作为用户主动设置的月份地图。

首版只要把“Memo 容器 + 内部待办 + 拖拽排序”做好，就已经足够清晰。
