# MemoTask V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build MemoTask V1 end to end from Phase 0 through Phase 5 according to `docs/memotask-v1-employee-task-plan.md`.

**Architecture:** Use a React/Vite PWA frontend and a Cloudflare Worker API. Keep domain logic in shared TypeScript services so it can be tested without Cloudflare, then adapt it to D1 in the Worker.

**Tech Stack:** React, TypeScript, Vite, Hono, Cloudflare Workers, D1, Vitest, Playwright, lucide-react, dnd-kit.

## Global Constraints

- UI language is Chinese.
- `docs/memotask-v1-employee-task-plan.md` is the source of truth.
- AI model default is `dsv4-pro`.
- Do not write full API keys to repo files, docs, committed code, screenshots, or logs.
- V1 must not include dates, Today, Upcoming, reminders, notifications, tags, independent Search page, payments, membership, subscriptions, upgrades, or self-built login.
- AI runs only from Capture via Analyze and writes editable Todo drafts before publish.
- Published Memo is not re-analyzed by AI.
- Todo belongs only to Memo.
- History stores whole Memo, not Todo流水.
- Complete Todo stays in place, checked and gray, without strikethrough.
- Cloudflare Access setup requires action-time confirmation before creating or modifying cloud policies.

## File Structure

- `package.json`: scripts, dependencies, test commands.
- `vite.config.ts`: frontend build and test config.
- `wrangler.toml`: Cloudflare Worker, assets, D1 binding.
- `migrations/0001_initial.sql`: D1 schema.
- `src/main.tsx`: React entry.
- `src/App.tsx`: routes and app shell.
- `src/styles.css`: Soft Clay Neumorphism design system.
- `src/types.ts`: shared client-side types.
- `src/api/client.ts`: API client.
- `src/state/app-state.ts`: data loading and mutation helpers.
- `src/components/*`: shell, nav, buttons, cards, forms, todo rows, toast.
- `src/pages/*`: Memos, Capture, MemoDetail, History, Settings.
- `worker/index.ts`: Worker/Hono app entry.
- `worker/db/schema.ts`: SQL statements and row mapping.
- `worker/db/d1-repository.ts`: D1 persistence.
- `worker/services/*`: Memo, Todo, History, AI settings, export, crypto, AI analyze.
- `worker/routes/*`: API route modules.
- `tests/*`: unit, API, and UI behavior tests.
- `docs/development-context.md`: long-running context anchor.
- `docs/cloudflare-setup.md`: Cloudflare setup/runbook updated during deployment work.
