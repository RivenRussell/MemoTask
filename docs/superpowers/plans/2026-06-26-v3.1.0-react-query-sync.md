# MemoTask v3.1.0 React Query Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship MemoTask v3.1.0 with React Query-based server state, faster login/startup, reliable PC/Android memo synchronization, a less rigid capture draft workflow, and release-ready desktop/APK artifacts.

**Architecture:** Keep Cloudflare Worker, D1, Electron, and Capacitor packaging intact. Refactor the frontend state layer so server data is owned by TanStack React Query while local UI state remains in React state. For v3.1.0, preserve the existing public App page contract and integrate React Query inside `useMemoTaskState` to minimize release risk; splitting it into route/auth/capture/memo/history/settings hooks is deferred until a later structural refactor.

**Tech Stack:** React 19, Vite, TypeScript, TanStack React Query, Cloudflare Worker/D1, Electron Builder, Capacitor Android, Vitest, Playwright.

## Global Constraints

- Version must be `3.1.0` in `package.json`, `package-lock.json`, README/version docs, desktop installer metadata, and Android package build outputs.
- PC app remains Electron packaging of the React build output and continues to request `https://memotask.rrwks.cn`.
- Android app remains Capacitor APK sideload packaging and continues to request `https://memotask.rrwks.cn`.
- Web/Worker remain deployable on Cloudflare Workers/Assets and D1.
- Do not introduce WebSocket/SSE in v3.1.0; use React Query cache invalidation, focus refetch, manual refetch, and low-frequency polling.
- Do not remove Android native share support or desktop tray/quick capture support.
- Preserve auth security: app clients keep bearer app session tokens; browser web keeps HttpOnly cookie behavior.
- Use TDD for production behavior changes: write failing tests first, verify red, implement, verify green.
- No speculative features beyond the requested sync, draft, login/startup, and release-readiness work.
