# MemoTask V2 Auth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add public email/password account management, password recovery, and per-user Memo isolation for MemoTask V2.

**Architecture:** The Cloudflare Worker owns auth through `/api/auth/*`, D1 auth tables, PBKDF2 password hashes, random token hashes, and HttpOnly session cookies. Existing Memo APIs become protected routes that resolve a verified user and pass `userId` into repository methods. React loads `/api/auth/me` before showing the app shell and renders auth screens when needed.

**Tech Stack:** React 19, TypeScript, Vite, Hono, Cloudflare Workers, Cloudflare D1, Web Crypto, Vitest, Testing Library.

## Global Constraints

- Public users can register with email/password and must verify email before entering the main app.
- Password reset uses email tokens and generic forgot-password responses.
- Sessions use HttpOnly cookies named `memotask_session`.
- All Memo, draft, history, AI settings, export, and sync status data must be isolated by authenticated `userId`.
- V2 does not include OAuth, teams, roles, admin dashboards, billing, public profiles, tags, reminders, or date/month map features.
- V1 baseline is preserved on `codex/memotask-v1`; V2 work lives on `codex/v2-auth`.
- Use TDD: write failing tests, confirm failure, implement minimal code, confirm pass.
